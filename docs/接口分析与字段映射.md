# 快递助手 ERP 接口分析与飞书映射

## 结论

现有接口能够支撑订单、售后、商品/SKU、成本、库存、店铺最终净利润和主播工资计算。工资提成直接采用 ERP 毛利报表的 `netSalesProfit`，退款、退货成本、实际成本和运费不在工资表重复计算。

客户持续维护的数据已经存在于飞书：

- `13_人员表`：姓名、所属店铺、基本工资、提成百分比。
- `14_工资表`：每月绩效工资、奖金、扣款。
- `15_工资规则说明`：全部业务规则，作为唯一生效口径。

## 接口与目标表

| 快递助手接口 | 可获取的主要数据 | 飞书目标表 | 同步唯一键 |
|---|---|---|---|
| `kdzs.erp.api.trade.list` | 店铺、订单号、下单/发货/修改时间、支付/实收、状态、商品、SKU、数量、退款状态 | `06_订单列表` | 系统订单号 + 子订单号 |
| `kdzs.erp.api.refund.list` | 售后单号、订单号、退款状态/金额、退货状态、数量、SKU、退款原因、退货物流 | `07_售后列表` | 退款单号 + SKU + 商品 + 规格 |
| `kdzs.erp.api.platform.item.list` | 平台商品、SKU、售价、编码、图片、上下架状态 | `11_平台商品` | 商品 ID + SKU ID |
| `kdzs.erp.api.sys.item.list` | ERP 商品/SKU、成本价、售价、条码、分类、供应商、品牌、重量 | `12_ERP货品` | 系统商品 ID + 系统 SKU ID |
| `kdzs.erp.api.stock.list` | 总库存、可配货库存、预占、在途、退货待处理 | `10_库存快照` | 系统 SKU ID |
| `kdzs.erp.api.report.logistics` | 订单号、运单号、快递公司编码、发货时间、物流状态、异常状态、处理状态 | `物流明细表` | 订单号 + 运单号 |
| `kdzs.erp.api.stock.in.list` | 入库单号、供应商、入库状态、入库数量、成本价、入库费用 | `库存入库表` | 入库单号 + SKU |
| `kdzs.erp.api.purchase.list` | 采购单、供应商、采购数量、已入库数量、采购金额、成本价 | `采购单表` | 采购单号 + SKU |
| `kdzs.erp.api.report.gross.profit` | 支付、收入、实际成本、运费、退款、退货成本、净销售额、最终净利润及利润率 | `02_店铺利润明细` | 日期 + 平台 + 店铺 |

毛利润报表的分组口径：`queryGroupType=2` 店铺、`7` 订单、`8` 订单商品。当前服务分别写入店铺利润、订单利润明细和商品利润明细。

## 售后和真实利润处理

- 仅退款、退货退款、换货、补差价、补发都由售后接口同步，不以订单表的一次性状态作为最终结论。
- 同一退款单的相同 SKU 会先合并再写入，避免重复扣减。
- 日常售后同步回看 45 天；店铺利润也回算 45 天，覆盖订单完成后的退款变化。
- 是否退回仓库、实际退货成本等影响由 ERP 最终毛利口径反映；工资计算只使用最终 `netSalesProfit`，不再次扣成本。
- 月结后保存利润快照和提成快照。晚于结算日确认的例外售后不篡改历史工资，按 `15_工资规则说明` 进入下月人工调整项。

## 客户维护项

| 数据 | 维护位置 | 默认值 |
|---|---|---|
| 员工对应店铺 | `13_人员表` | 必填 |
| 基本工资 | `13_人员表` | 必填 |
| 提成百分比 | `13_人员表` | 必填 |
| 绩效工资 | `14_工资表` | 0 |
| 奖金 | `14_工资表` | 0 |
| 扣款 | `14_工资表` | 0 |

平台服务费、投流费用等无需客户重复提供：当前规则明确认可 ERP 返回的最终店铺净利润。如果将来客户改为自行拆分利润口径，才需要另行增加这些费用来源。

## 建议同步频率

| 数据 | 推荐频率 | 原因 |
|---|---|---|
| 订单、售后 | 每 1–2 小时或每天至少一次 | 捕获订单状态和退款变化 |
| 库存 | 每 2 小时 | 经营看板及时性 |
| 商品/SKU、成本 | 每天一次 | 变化频率较低 |
| 店铺利润 | 每天凌晨回算近 45 天 | 以稳定口径吸收退款和成本变化 |
| 工资结算 | 每月 15 日，利润同步成功后 | 防止不完整数据进入工资 |

当前服务按小时同步订单、售后、物流和库存；每日补同步商品、SKU、采购/入库和利润。历史全量回填使用 `new-backfill`，按7天订单/物流窗口和逐日利润窗口执行，避免快麦接口大时间范围报错。
